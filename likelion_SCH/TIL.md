## 2주차 📖 TIL 📖


  ● 1. 오늘 배운 개념/주제
    
    ○ ML의 개념과 학습 방법(지도, 비지도, 강화 학습), 각 학습 방법의 종류에 대해 배움

    ○ 모델을 학습시키기 전 데이터 전처리 방법
  
  ● 2. 핵심 내용 정리

    ○ 지도 학습은 컴퓨터에게 데이터와 라벨을 모두 제공하고 도출 방법을 학습하는 방법
        → 종류: 회귀(Regression)은 연속적인 수치, 분류(Classification)은 불연속적인 카테고리를 예측함 
            ▶ 회귀 방법론 
              ■ K-최근접 이웃 회귀(KNN): 예측하려는 새로운 데이터 포인트에 가장 가까이 있는 기존 데이터 K개를 탐색 후에 결과값의 평균
              ■ (단순, 다중)선형 회귀: 분산된 데이터의 분포 속에서 가장 그럴듯한 직선 하나를 찾는 방법 (OLS, GLS, 경사 하강법)
            ▶ 분류 방법론
              ■  로지스틱 회귀: 선형 회귀법의 결과값의 범위를 [0,1] 사이의 확률값으로 변환 (확률적 경사 하강법)
              ■  나이브 베이즈: 통계학의 베이즈 정리를 기반, 모든 변수를 독립적이라 가정하고 조건부 확률을 구함
              ■  결정 트리: 데이터를 잘 분리할 수 있는 질문을 찾아 나뭇가지 형태로 데이터를 쪼개 나감 - 과적합 위험 존재
              ■  랜던 포레스트: 결정 트리와 이들의 예측 결과를 다수결로 종합하여 최종 결과를 내는 앙상블 기법 
              ■  XGBoost: 이전 트리의 오차를 다음 트리가 이어받아 학습 후 오차를 줄여가는 앙상블 기법(Boost)을 표방한 알고리즘
              ■  서포트 벡터 머신(SVM): 서로 다른 특징을 가진 데이터의 집합을 가장 효과적으로 나눌 수 있는 초평면(결정 경계 선) 을 찾는 알고리즘
   
    ○ 비지도 학습은 컴퓨터에게 라벨 없이 데이터만 제공하는 방법
        → 비슷한 성향의 데이터를 묶는 Clustering, 복잡하고 불필요한 데이터의 차원을 줄이는 Dimension Reduction이 활용 방법
            ▶ 비지도 학습 방법론
              ■ K-Means: 각 그룹의 개수(K)에 맞게 중심점을 배치 후 각 데이터가 중심점에 가장 잘 들어맞을 때까지 중심점을 이동시키는 방법
              ■ 주성분 분석(PCA): 데이터가 가장 넓게 퍼져 있는 방향을 축으로 지정한 후 차원을 축소하는 방법

    ○ 강화 학습은 에이전트(컴퓨터)가 주어진 환경에서 어떤 행동을 취하고 그에 대한 보상이나 벌점을 받으며 학습하는 방법
        → 행동을 수없이 많이 반복하면서 보상을 최대화하는 최적의 행동 순서를 스스로 학습하도록 하는 것이 목적

    ○ 데이터 전처리는 컴퓨터가 데이터의 정답만 외우는 과적합(Overfitting)을 예방하기 위해 꼭 실시하는 과정
        → 모델이 처음 보는 데이터에도 잘 대응하는 지 평가하기 위해 테스트용 데이터와 학습용 데이터를 꼭 분리해놓아야 함!!

    ○ 모든 학습 방법이나 데이터 전처리는 사이킷런(sklearn) or Pandas를 통해 불러옴

## 3주차 📖 TIL 📖


  ● 1. 오늘 배운 개념/주제
      
    ○ 데이터 전처리에 대한 자세한 내용 학습

  ● 2. 핵심 내용 정리

    ○ 결측치란 값이 누락된 데이터를 의미
      
      ▶ 처리 방법 1: 결측치 확인 → 결측치 제거 (dropna) - 빈칸이 포함된 행이나 열을 아예 삭제하는 방법
                      데이터의 총량이 많거나 결측치의 비율이 매우 작을 때 사용
      
      ▶ 처리 방법 2: 결측치 확인 → 결측치 대체 (fillna) - 빈칸을 다른 값으로 채워 넣는 방법
                      
                      평균값 - 정규분포를 따르는 데이터에서 이용
                      중앙값 - 데이터에 이상치(outliers)가 있을 때 사용
                      최빈값 - 카테고리로 나뉘는 데이터에서 사용

    ○ 범주형 데이터 인코딩 (Categorical Encoding) 이란 모든 텍스트 데이터를 숫자로 변환해 주는 작업

      ▶ 라벨 인코딩: 카테고리마다 고유의 번호를 부여 /AI가 숫자의 크기를 가치나 순위로 오해할 수 있으므로 트리 기반 모델에서만 사용 권고

      ▶ 원-핫 인코딩: 데이터가 가진 카테고리 종류만큼 열을 추가 후 해당하는 열은 1, 나머지는 모두 0으로/ 카테고리가 너무 많으면 사용하지 않는 것이 좋음


    ○ 데이터 스케일링은 압도적으로 큰 숫자를 비슷한 범위로 압축해 줘야 함 (log scale로 변환한다 생각)

      ▶ 최소 최대 스케일링(Min-Max Scaler): 최솟값 0, 최댓값을 1로 고정/ 이상치를 정상 데이터와 비슷하게 맞추기 힘듦

      ▶ 표준화(Standard Scaler): 데이터의 평균을 0, 표준편차를 1로 변환/ 정규분포를 따른다고 가정하는 알고리즘에 유용하고 이상치에 비교적 덜 빈감

    ❗데이터 스케일링 유의점❗훈련 데이터를 스케일링 할 때는 .fit_transform()을 사용하지만 시험 데이터를 스케일링 할 때는 .transform()만 사용해야 함 → 미지의 데이터의 정답을 맞춰야 하는 AI에게 힌트를 주는 것과 같은 꼴


    ○ 이상치(오타형 이상치, 평균에 지나치게 벗어나는 이상치) 탐지과 제거 

      ▶ boxplot을 통해 그래프를 그려 이상치 확인

      ▶ IQR: 데이터를 4분위로 나누어 2분위와 3분위 사이의 핵심 구간을 기준으로 삼아 이상치를 찾아내는 방법

## 4주차 📖 TIL 📖

  ● 1. 오늘 배운 개념/주제
      
    ○ 머신러닝과 딥러닝
    ○ 인공신경망 (입력층, 은닉층, 출력층)
    ○ 텐서, 배치사이즈, 에포크, 손실 함수, 옵티마이저, 역전파 (backpropagation), 학습률
    ○ 딥러닝 모델의 종류 - RNN/LSTM, Transformer, GAN, AutoEncoder 
    ○ MNIST 

  ● 2. 핵심 내용 정리

    ○ 머신러닝은 사람이 힌트를 줘야 하는 AI이고 딥러닝은 스스로 눈을 키우는 AI이다
    ○ 인공신경망은 인간의 뇌구조를 따라한 AI의 뇌구조라고 보면 된다
      → 입력층: 데이터가 들어오는 문
      → 은닉층: 데이터 속의 패턴을 찾는 곳, 이 층이 여러 개라면 딥(deep)러닝이라 부름
      → 출력층: 최종 답을 내는 곳
      → 각 뉴런은 정보의 중요도에 따라 가중치를 매김
      → 활성화 함수: 어떤 뉴런이 반응해야 하는지 결정하는 스위치 ex) ReLU (음수면 0, 양수면 통과)
                     복잡한 패턴을 찾기 위해서는 반드시 필요한 함수
    ○ 텐서: 3차원 이상의 행렬 → 데이터를 담는 그릇이라 생각
            PyTorch에서는 모든 데이터를 텐서 형태로 다룸 
            Numpy의 ndarray와 비슷하지만 PyTorch는 GPU에서 직접 연산할 수 있음
      
      배치사이즈: 전체 데이터를 몇으로 나눠 학습? → 보통 32,64,128 중 하나를 씀

      에포크: 전체 데이터를 1번 완전히 학습시켰을 때의 단위 → 1에포크 / 보통 5~50번 정도 반복

      손실 함수: 얼마나 틀렸는지 점수 매기기 → 손실 값을 최대한 줄이는 것 목표
                → 분류(0~9 중 맞추기) / CrossEntropyLoss 사용 → 정답 확률을 얼마나 높게 줬느냐가 기준
                → 회귀(숫자 예측) / MSELoss 사용 → 분산

      옵티마이저: 손실이 가장 작은 지점을 어떻게 도달하는 지를 개선
                → SGD / 눈 앞의 데이터에만 집중, 단순하지만 느린 방법
                → Adam/ 지금까지의 경험을 기억, 효율적이라 가장 많이 쓰이는 방법

      역전파: 어디서 틀렸는지 추적
              → 아래의 단계 수천 ~ 수만 번 반복하여 정답에 근접
              
                순전파     : 입력 → 모델이 예측
                손실 계산  : 예측 vs 정답 비교 → "얼마나 틀렸나?"
                역전파     : 거꾸로 추적 → "어떤 가중치 때문에 틀렸나?"
                업데이트   : 문제가 된 가중치를 조금 수정

      학습률: 옵티마이저가 손실 최저점에 도달하는 보폭 
             → 너무 크면 최적점을 지나치며 발산
             → 너무 작으면 학습이 느림
             → 보통 0.001로 시작 

    ○ 딥러닝 종류
      
       → CNN (Convolutional Neural Network) : 이미지 위주의 딥러닝 / 활용: 사진 분류, 자율주행 물체 인식 
       → RNN/LSTM: 데이터의 순서와 흐름을 기억하면서 처리 / 활용: 주가 예측, 음성 인식
       → Transformer: 현재 AI의 핵심, 어디가 중요한지 스스로 판단하는 주의집중(Attention) 메커니즘 / 활용: ChatGPT, Claude, 번역, 이미지 생성 등 거의 모든 분야
       → GAN (Generative Adversarial Network): 생성자(위조)와 판별자(감별)라는 독특한 매커니즘, 둘을 경쟁시켜 생성자의 위조 실력 향상/ 활용: 얼굴 생성, 사진 스타일 변환, 가짜 데이터 생성
       → AutoEncoder: 데이터를 최대한 작게 압축했다가 원래대로 복원하는 구조, 노이즈 제거나 이상 탐지에 특화/ 활용: 결함 감지, 노이즈 제거, 추천 시스템

    ○ MNIST:  0~9까지 손으로 쓴 숫자 이미지 7만 장으로 구성된 데이터셋
              각 이미지는 28×28 픽셀 흑백 이미지
